# MINLAB 3D

## Цель проекта

Веб-приложение для автоматизации учёта пластика и расчёта стоимости заказов в студиях 3D-печати. Помогает печатникам и студиям контролировать остатки филамента, быстро рассчитывать себестоимость изделий, вести CRM заказов, отслеживать финансовые показатели и обмениваться опытом в ленте постов.

## Основные возможности

### Авторизация и профили

- **Цель:** Пользователь хочет иметь личный профиль, а доступ к функциям приложения — только после входа.
- **Описание:** Простая авторизация по имени и паролю. У каждого пользователя свой профиль с именем. Данные (катушки, заказы, настройки) привязаны к пользователю. При первом входе создаётся аккаунт.
- **Критерии приёмки:**
  - При запуске приложения показывается экран входа
  - Можно зарегистрироваться (имя + пароль) или войти в существующий аккаунт
  - После входа пользователь видит основное приложение со своими данными
  - Выход из аккаунта возвращает на экран входа

### Лента постов (Feed)

- **Цель:** Пользователи могут публиковать гайды, советы и файлы для печати, а также просматривать посты других, комментировать и редактировать свои.
- **Описание:** Отдельная вкладка с лентой постов в стиле соцсети. Посты содержат текст, можно прикрепить файлы. Отображается автор и время публикации. Автор может редактировать или удалить свой пост. Под постами можно писать комментарии. Посты и комментарии видны всем пользователям приложения.
- **Критерии приёмки:**
  - Лента отображается как отдельная вкладка в боковом меню
  - Посты показываются в хронологическом порядке (сначала новые)
  - У каждого поста виден автор и дата/время публикации
  - Можно создать новый пост с текстом и прикрепить файл
  - Автор может редактировать текст своего поста
  - Автор может удалить свой пост
  - Под каждым постом можно писать комментарии
  - Комментарии сохраняются в LocalStorage

### Учёт катушек с пластиком (Filament Stock)

- **Цель:** Пользователь хочет вести склад филамента: добавлять катушки, отслеживать остатки, удалять закончившиеся.
- **Описание:** Сетка карточек катушек с визуальным прогресс-баром остатка. При добавлении катушки начальный вес выбирается из списка (1000г, 2000г, 3000г, 5000г). При остатке менее 150г — жёлтое предупреждение. При нулевом остатке — статус «Закончился». Можно удалить катушку. Вес автоматически списывается при переводе заказа в статус «Готов». При добавлении катушки цвет выбирается из RGB-палитры.
- **Критерии приёмки:**
  - Можно добавить новую катушку с указанием производителя, типа пластика, цвета (RGB-палитра), начального веса (выбор из 1000/2000/3000/5000г) и цены
  - Каждая катушка отображается в виде карточки с прогресс-баром остатка
  - При остатке < 150г карточка подсвечивается жёлтым индикатором
  - При остатке 0 катушка помечается как закончившаяся
  - Можно удалить катушку со склада
  - При переводе заказа в статус «Готов» вес изделия автоматически вычитается из остатка катушки

### Калькулятор себестоимости заказа

- **Цель:** Пользователь хочет мгновенно рассчитать себестоимость изделия и конечную цену для клиента.
- **Описание:** Форма с полями: масса изделия (г), время печати (ч), выбор катушки из склада, галочка учёта постобработки. Мгновенный расчёт себестоимости пластика, стоимости работы (время × тариф электроэнергии + амортизация), стоимости постобработки (если включена), общей себестоимости, маржинальности и чистой прибыли. Поле «Окончательная стоимость» — по умолчанию равна итоговой цене, пользователь может увеличить, но не может поставить меньше итоговой. Кнопка «Оформить как заказ» передаёт данные в CRM. Нельзя оформить заказ, если масса изделия превышает текущий остаток выбранной катушки.
- **Критерии приёмки:**
  - Все поля ввода работают и обновляют расчёты мгновенно при изменении
  - Себестоимость пластика = масса изделия × (цена катушки / начальный вес)
  - Стоимость работы = время печати × тариф электроэнергии + амортизация (из настроек)
  - Галочка постобработки включает/отключает блок настройки постобработки
  - Отображается чистая прибыль и маржинальность в %
  - Окончательная стоимость не может быть меньше итоговой цены
  - Нельзя оформить заказ, если масса изделия больше остатка выбранной катушки
  - Кнопка «Оформить как заказ» создаёт заказ и переключает на вкладку CRM

### CRM и история заказов

- **Цель:** Пользователь хочет управлять заказами: создавать, менять статусы, отслеживать историю.
- **Описание:** Таблица заказов с фильтром по статусам. Статусы в виде цветных бейджей: «Принят», «Готов», «Выдан». Статус меняется через выпадающий список. При переводе в статус «Готов» автоматически списывается масса изделия со склада.
- **Критерии приёмки:**
  - Отображается таблица со всеми заказами: ID, клиент, изделие, пластик, себестоимость, цена, статус, дата
  - Статусы отображаются цветными бейджами: «Принят», «Готов», «Выдан»
  - Статус меняется через выпадающий список (select)
  - При смене статуса на «Готов» масса изделия списывается с выбранной катушки
  - Работает фильтрация заказов по статусам

### Дашборд аналитики

- **Цель:** Пользователь хочет видеть ключевые финансовые показатели и критические остатки на одном экране.
- **Описание:** Карточки KPI (выручка, прибыль, активные заказы), блок критических остатков (< 150г) с красной подсветкой, простые CSS-графики доходов за текущий месяц.
- **Критерии приёмки:**
  - Отображается общая выручка, чистая прибыль, количество активных заказов
  - Список катушек с остатком < 150г выделен красным
  - График доходов за текущий месяц отображается корректно

### Глобальные настройки

- **Цель:** Пользователь задаёт базовые параметры для расчётов.
- **Описание:** Модальное окно с полями: тариф электроэнергии (за час), амортизация оборудования (за час). Валюта только BYN (белорусский рубль). Значения подтягиваются в калькулятор себестоимости.
- **Критерии приёмки:**
  - Настройки сохраняются и применяются во всех расчётах
  - Тариф электроэнергии и амортизация предзаполнены значениями по умолчанию
  - Валюта только BYN, переключение валюты отсутствует

### Учёт постобработки

- **Цель:** Пользователь хочет добавлять стоимость постобработки (краски, грунтовка, гравировка) как дополнительную плату к заказу.
- **Описание:** В калькуляторе стоит галочка «Постобработка». Если включена — появляются поля для настройки: выбор изделия (из заказов), стоимость постобработки. Сумма прибавляется к себестоимости и учитывается в расчёте конечной цены и маржинальности.
- **Критерии приёмки:**
  - Галочка «Постобработка» включает/отключает блок настройки
  - Можно выбрать изделие из существующих заказов
  - Сумма постобработки учитывается в общей себестоимости
  - При оформлении заказа стоимость постобработки сохраняется в данных заказа

## Дополнительные заметки

- Тёмная индустриальная тема (dark mode по умолчанию) с возможностью переключения на светлую
- Модульная архитектура: боковое меню (Sidebar) для переключения вкладок
- Хранение данных в LocalStorage — приложение работает сразу в браузере без сервера
- Валюта: BYN (белорусский рубль)
- Сценарии первой версии: авторизация, склад пластика, калькулятор себестоимости, CRM заказов, дашборд аналитики, настройки, учёт постобработки, лента постов
